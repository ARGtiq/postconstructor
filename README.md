# Конструктор Гарипова

Веб-конструктор карусели для медицинского контента (@dr.garipov): слайды, темы оформления, экспорт в PNG/ZIP, расписание публикаций, библиотека проектов с опциональной синхронизацией между устройствами.

Работает как обычная веб-страница и как устанавливаемое PWA-приложение (офлайн после первого захода).

## Структура репозитория

```
index.html                  — приложение целиком (разметка + вся логика)
manifest.json                — PWA-манифест (иконка, имя, standalone-режим)
sw.js                        — service worker (network-first)
icons/                       — иконки PWA (192, 512, maskable 512)
supabase_schema.sql          — SQL-схема таблицы + RLS-политики для синхронизации
```

## Как открыть

**Просто как файл:** дважды кликнуть `index.html`. Работает полностью, кроме автосохранения в файл на диске (требует HTTPS) и PWA-установки.

**Как сайт (рекомендуется):** GitHub Pages → см. раздел ниже. Тогда доступны PWA-установка и полный набор функций.

## Хранение данных

Библиотека проектов хранится:
- в браузере (`localStorage`) — по умолчанию, без дополнительной настройки;
- в облаке (Supabase) — опционально, для синхронизации между устройствами (см. ниже).

Экран **«Проекты»** внизу приложения — точка входа: сохранение текущей карусели в библиотеку, открытие/переименование/удаление сохранённых проектов, ручной экспорт/импорт бэкапа в JSON-файл.

## Синхронизация между устройствами (опционально)

Приложение личное — синхронизация нужна, только если редактируешь на нескольких устройствах (например, начал на компьютере, продолжил на телефоне). Реализована через [Supabase](https://supabase.com) (бесплатный план) с входом по email-ссылке (Magic Link), без паролей.

### Настройка (один раз)

1. Создать проект на supabase.com.
2. **SQL Editor** → вставить содержимое `supabase_schema.sql` → Run. Это создаёт таблицу `projects` и включает Row Level Security — каждый пользователь видит и может менять только свои строки.
3. **Authentication → Providers → Email** → включить Magic Link.
4. **Authentication → URL Configuration → Site URL** → указать адрес, по которому открывается приложение (например, `https://username.github.io/repo-name/`).
5. **Settings → API** → скопировать `Project URL` и ключ `anon public`.
6. В приложении: **Настройки → Синхронизация** → вставить эти два значения → «Сохранить».

Ключи хранятся только в `localStorage` этого браузера/устройства — они **не** попадают в код приложения и не публикуются вместе с репозиторием. На каждом новом устройстве шаг 6 нужно повторить (или синхронизировать сами ключи вручную — они не секретные, но лучше не хранить в открытом репозитории на всякий случай).

### Использование

Экран **«Проекты»** → «Войти» → email → перейти по ссылке из письма → кнопка «Синхронизировать сейчас». Слияние идёт по принципу «новее побеждает» (сравнение по времени последнего изменения), без потери записей — конфликтующие изменения не удаляются молча.

### Безопасность и лимиты (коротко)

- Anon-ключ не секретный сам по себе — доступ к данным контролирует Row Level Security на сервере, а не секретность ключа.
- Приложение не предназначено для хранения данных пациентов — только контент публикаций.
- Бесплатный план Supabase ставит неактивные (без обращений неделю) проекты на паузу; при следующем открытии синхронизация может не сработать, пока проект не «разбудить» в панели Supabase.
- Изображения внутри карусели хранятся как часть JSON (base64) — при большом количестве стоит следить за лимитом базы (500 МБ на бесплатном плане).

## GitHub Pages

1. Загрузить в репозиторий: `index.html`, `manifest.json`, `sw.js`, папку `icons/`.
2. Settings → Pages → Source: ветка `main`, папка `/root`.
3. Через пару минут приложение доступно по `https://username.github.io/repo-name/`.
4. Указать этот адрес в Supabase (см. шаг 4 выше), если используется синхронизация.

## PWA-установка

На телефоне: открыть ссылку → меню браузера → «Добавить на главный экран».
На компьютере (Chrome/Yandex Browser): иконка установки в адресной строке.

После установки приложение открывается отдельным окном без адресной строки браузера и работает офлайн (после первого захода в сеть).


## Известные ограничения

- Синхронизация — не «живая» (не в реальном времени), запускается вручную кнопкой. Для личного использования с одним активным устройством за раз этого достаточно.
- Масштаб некоторых визуальных эффектов (blur, паттерны) при экспорте PNG в форматах «Сторис»/«Рилс» рассчитан на ту же геометрию, что и формат «Пост»; для вертикальных форматов возможно небольшое визуальное расхождение с превью — под уточнение.
